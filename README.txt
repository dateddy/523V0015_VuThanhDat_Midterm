  Skip-gram Word2Vec — From Scratch
  NLP midterm | Tasks 1–4

OVERVIEW
--------
This midterm project implements a Skip-gram Word2Vec model from scratch using NumPy,
trains it on a small hand-crafted corpus, evaluates the learned word embeddings,
and compares the results against Gensim's Word2Vec implementation.

FILES
-----
  skipgram.py     — Core implementation (corpus, model, training, experiments)
  evaluate.py     — Evaluation: cosine similarity, nearest neighbours, Gensim
  loss_curve.png  — Training loss curve generated by Task 3.2
  README.txt      — This file


REQUIREMENTS
------------
  Python  >= 3.8
  numpy
  scipy
  matplotlib
  gensim          (optional — required only for Task 4.3 Gensim comparison)

Install dependencies:
  pip install numpy scipy matplotlib gensim


HOW TO RUN
----------
1. Run the core Skip-gram module (Tasks 1–3):
     python skipgram.py

   This will:
   - Print corpus preprocessing stats (vocab size, pair counts)
   - Run forward/backward pass verification
   - Train the baseline model and print checkpoint losses
   - Save loss_curve.png to the current directory
   - Run and print hyper-parameter experiment results

2. Run the evaluation module (Task 4):
     python evaluate.py

   This will:
   - Re-train the baseline model
   - Compute and display cosine similarities for 6 word pairs
   - Display top-3 nearest neighbours for 4 query words
   - (If gensim is installed) Compare results against Gensim Word2Vec


PROJECT STRUCTURE BY TASK
--------------------------
  Task 1 — Corpus Preprocessing
    - Tokenisation, vocabulary construction, word2idx / idx2word mappings
    - Sliding-window (W=2) context pair generation

  Task 2 — Skip-gram Model
    - Numerically stable softmax and cross-entropy loss
    - SkipGram class: forward pass, backward pass (SGD)
    - Gradient checking (numerical vs analytical)

  Task 3 — Training
    - 3.1  Baseline training: d=10, η₀=0.025, decay=0.005, T=100 epochs
           Expected losses — Ep10: ~3.257 | Ep50: ~2.573 | Ep100: ~2.041
    - 3.2  Learning curve plot saved to loss_curve.png
    - 3.3  Hyper-parameter experiments:
             Experiment A: d=20 (double embedding dimension)
             Experiment B: fixed η=0.05 with no learning-rate decay

  Task 4 — Evaluation
    - 4.1  Cosine similarity between selected word pairs
    - 4.2  Top-k nearest-neighbour retrieval using W_in embeddings
    - 4.3  Gensim comparison: extended similarity table, Spearman rank
           correlation, and two technical differences (full softmax vs
           negative sampling; sub-sampling of frequent words)


BASELINE HYPER-PARAMETERS
--------------------------
  Embedding dimension   d  = 10
  Initial learning rate η₀ = 0.025
  LR decay schedule        = η₀ / (1 + 0.005 × epoch)
  Epochs                T  = 100
  Random seed              = 0
  Weight init scale        = 0.01


NOTES
-----
- All embeddings are taken from W_in (the input / center-word matrix).
- Training pairs include duplicates (not de-duplicated before training)
  to reflect true co-occurrence frequency in the corpus.
- Gensim comparison (Task 4.3) is skipped gracefully if gensim is not
  installed; all other tasks will still run correctly without it.
